# bakewiki 설계 문서

> 최종 수정: 2026-06-27

## 1. 개요

**bakewiki**는 LLM과 인간 모두를 1등 시민으로 대하는 오픈소스 위키입니다.
콘텐츠는 GFM(GitHub Flavored Markdown)으로 작성되며, 향후 호스팅 서비스도 제공할 예정입니다.

### 핵심 가치
- **인간과 LLM 모두 1등 시민**: 인간이 읽고 편집하기 좋은 UI와 동시에 LLM이 소비하기 좋은 구조화된 API를 제공
- **GFM 기반**: GitHub Flavored Markdown을 표준 포맷으로 사용
- **오픈소스**: AGPLv3 라이선스

---

## 2. 데이터 모델

### 2.1 식별자와 경로
- **슬러그 = 디렉토리 + 타이틀**: 파일 시스템 스타일의 계층 경로 사용 (`tech/web/HTTP`)
- **타이틀 = 첫 `#` 헤딩**: 본문 첫 번째 `#` 헤딩이 페이지 제목. frontmatter `title` 필드 없음
- **슬러그 유도**: 새 문서 생성 시 디렉토리 + 첫 `#` 헤딩에서 자동 유도 (`# HTTP` → `HTTP`)
- **유니코드 지원**: 한글 등 유니코드 슬러그 허용 (`히히`, `파일들`)
- **`index` 특수 슬러그**: 홈페이지는 항상 `index`
- **다국어 매핑 기능**: 없음 (별개 문서로 취급)

### 2.2 메타데이터
- **Frontmatter**: YAML 메타데이터 지원
  - 유일한 필드: `public` (boolean, 기본값 `true`)
  - `title` 필드는 사용하지 않음 (본문 첫 `#` 헤딩으로 대체)

### 2.3 링크
- GFM 표준 링크만 사용 (`[텍스트](경로)`)
- **절대 경로**: `/`로 시작 → 루트 기준 (`[HTTP](/tech/web/http)`)
- **상대 경로**: 부모 디렉토리 기준 (표준 URL 해석)
  - 슬러그 `tech/web/HTTP`에서 `CSS` → `tech/web/CSS` (형제)
  - 슬러그 `tech/web/HTTP`에서 `../API` → `tech/API` (삼촌)
  - 슬러그 `tech/web/HTTP`에서 `./HTTP/HTTPS` → `tech/web/HTTP/HTTPS` (자식)
- 위키링크(`[[문서명]]`) 문법은 미지원
- 트랜스클루전(`![[slug]]`) 미지원

## 3. 기술 스택

| 영역 | 선택 |
|------|------|
| 백엔드 프레임워크 | **Hono** (초경량, `npx` 단일 실행에 적합) |
| 언어 | Node.js + TypeScript |
| 데이터베이스 | SQLite (FTS5 내장) |
| ORM | **Drizzle ORM** (타입 안전, 마이그레이션 지원) |
| 검색 | SQLite FTS5 |
| 마크다운 파서 | **markdown-it** (빠르고 단순, 플러그인 생태계) |
| 프론트엔드 | SSR (서버 사이드 렌더링) |
| 템플릿 엔진 | **Handlebars** (`{{ }}` 문법, 로직리스) |
| 에디터 | **Milkdown** (WYSIWYG 마크다운 에디터, 실시간 미리보기 내장) |
| 코드 하이라이팅 | **highlight.js** (가볍고 SSR 친화적) |
| 수식 렌더링 | **KaTeX** (빠르고 SSR 친화적) |
| 비밀번호 해시 | **bcrypt** (검증된 업계 표준) |
| 세션 관리 | **쿠키 기반 세션** (서버 DB 저장, 즉시 폐기 가능) |
| 패키지 구성 | 단일 npm 패키지 |
| 소스 구조 | **플랫** (`src/` 아래 파일 나열, 단순) |
| 라이선스 | AGPLv3 |

## 4. 접근 권한

### 4.1 읽기
- **공개/비공개 문서 혼합**: 문서별로 공개 여부(frontmatter `public`) 설정
- 인증 없음 → 공개 문서만 조회
- 인증 있음 (API 키 또는 세션) → 모든 문서 조회

### 4.2 편집
- **단일 관리자만 편집 가능** (`init`에서 생성한 관리자 1명)
- 관리자는 이메일/비밀번호로 인증
- 관리자를 신뢰하므로 마크다운 렌더링 시 별도 Sanitize 처리 없음

### 4.3 인증
- **이메일/비밀번호**: 자체 계정 시스템 (웹 UI용, 단일 관리자)
- **API 키**: LLM/스크립트용 토큰 기반 인증
- **세션**: 쿠키 기반 세션 (서버 DB에 저장, 즉시 폐기 가능)

## 5. API (REST)

### 5.1 핵심 엔드포인트

| 메서드 | 경로 | 설명 | 인증 |
|--------|------|------|------|
| GET | `/api/pages/:slug` | 문서 조회 (원문 GFM + HTML + frontmatter) | 공개문서는 무인증 |
| GET | `/api/pages` | 문서 목록 (경로/네임스페이스별) | 공개문서만 |
| GET | `/api/search?q=` | 전문 검색 (FTS5) | 공개문서만 |
| POST | `/api/pages/:slug` | 문서 생성/수정 (전체 콘텐츠) | 관리자 |
| PATCH | `/api/pages/:slug` | 부분 업데이트 (public/body/slug) | 관리자 |

### 5.2 LLM 친화적 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/pages/:slug?format=plain` | HTML 없이 순수 텍스트/frontmatter만 반환 (토큰 절약) |
| GET | `/api/sitemap` | 전체 문서 구조를 트리/목록으로 반환 (LLM이 전체 맵 파악) |

### 5.3 페이지네이션
- **없음**: 목록/검색 결과 전체 반환 (MVP 단순화)

## 6. CLI (`npx bakewiki`)

| 서브커맨드 | 설명 |
|------------|------|
| `init` | 현재 디렉토리에 프로젝트 초기화 (DB 생성, 설정 파일 생성, **인터랙티브 프롬프트로 단일 관리자 계정 생성**) |
| `serve` | 로컬 HTTP 서버 부팅 (예: `http://localhost:3000`). 시작 시 자동 마이그레이션 |
| `import <dir>` | 로컬 마크다운 폴더 → DB 동기화 (초기 데이터 씨드/마이그레이션용) |
| `export <dir>` | DB → 로컬 마크다운 폴더 동기화 (백업/마이그레이션용) |
| `version` | 버전 확인 |

### 환경변수 (`.env` 파일 + 환경변수)

---

## 8. 호스팅

- **MVP**: 싱글테넌트 (하나의 인스턴스 = 하나의 위키)
- **향후**: 멀티테넌트 확장 고려하여 데이터 모델/구조 설계
  - 여러 사용자가 각자의 위키를 운영할 수 있도록 (서브도메인 또는 경로 분리)

---

## 9. 프로젝트 구성

- **단일 npm 패키지**: 서버, 프론트엔드, API, CLI가 하나의 패키지에 통합
- **플랫 소스 구조**: `src/` 아래 파일을 나열 (단순, MVP에 적합)
- `npx bakewiki`로 즉시 실행 가능
- **라이선스**: AGPLv3
